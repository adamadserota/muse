SunoBrain

Suno v5.5 Production Studio

SunoBrain is an AI-powered production blueprint generator that transforms ideas into fully optimized output for the Suno v5.5 "chirp-crow" engine. It produces style prompts, negative prompts, structured lyrics with performance metatags, deep musical analysis, and AI-generated album artwork — everything needed to create a studio-quality track in Suno.

How It Works — Overview

At its core, SunoBrain sits between you and Google Gemini 3.1 Pro. You provide creative input — a theme, existing lyrics, genre selections, artist references, or any combination — and SunoBrain routes it through a specialised prompt engineering system that instructs Gemini to produce output precisely formatted for Suno v5.5.

The system never generates audio. It produces text blueprints that you paste into Suno's interface. Each blueprint contains four production-facing outputs, plus an optional AI-generated album cover.

Output 1

Style Prompt

A comma-separated set of sonic descriptors (max 1,000 chars) that controls genre, vocals, instrumentation, production quality, tempo, and key. This is pasted into Suno's "Style of Music" field.

Output 2

Negative Prompt (Exclude Styles)

Contextually-derived terms that Suno should avoid — protecting the track's sonic identity from unwanted artifacts, genre conflicts, and production degradation.

Output 3

Optimized Lyrics

A full lyric sheet structured with Suno v5.5 metatags — section markers, performance cues, vocal delivery instructions, and instrumental behaviour tags — targeting a 3:00–3:30 radio edit.

Output 4

Musical Analysis

Deep analytical breakdown: Vibe DNA (emotional fingerprint), Phonetic Mapping (consonant/vowel rhythm), and Semantic Weight (narrative peak distribution).

Generation Modes

SunoBrain offers two top-level input modes, each with sub-flows. The mode determines which Gemini prompt template is used and how your input is processed.

One-shot Mode

A single text input that accepts anything — a theme description, existing lyrics, a URL, a single word, an artist name, or a concept. Gemini analyses the input, determines what it is, and produces the full production blueprint in one step.

Backend Mode theme_oneshot

The prompt instructs Gemini: "You will receive user input that could be anything — a theme description, existing lyrics, a URL, a single word, or a concept. Analyze the input, determine what it is, and produce the appropriate Suno v5.5 production blueprint."

Builder Mode

A structured multi-field interface with five optional input sections. Users fill in whichever sections they want — from just selecting genres to providing a full set of inspiration, lyrics, style preferences, and exclusions. Builder mode supports both one-shot and two-step generation flows.

Flow Backend Mode What Happens
Builder → One-shot builder_oneshot All builder inputs compiled and sent to Gemini. Returns full blueprint (styles, exclude, lyrics, analysis) in one call.
Builder → Two-step builder_draft then optimize_draft Step 1: Gemini generates raw draft lyrics (no metatags). User reviews and edits. Step 2: Edited draft is sent for full optimization — metatags, styles, exclusions, and analysis are generated around the user's words.

Builder Input Sections

In Builder mode, the frontend compiles all non-empty sections into a single delimited text block before sending to the API. Each section has strict semantic boundaries that the prompt enforces.

Section A

General Inspiration

Mood, vibe, concept, or theme. Influences all outputs — treated as the soul of the track. This is the broadest creative direction and affects style, lyrics, exclusions, and analysis equally.

Example: "A melancholic yet hopeful song about leaving your hometown, with imagery of autumn leaves and empty train stations"

Section B

Style Selector (Genre Multi-Select)

30 predefined genres selectable as chips. Influences the style prompt only, not lyrics. Genres are never listed verbatim — Gemini derives specific subgenre production techniques, instrumentation, and sonic textures unique to each combination.

Multi-selecting creates fusion: selecting "DnB + Jazz" produces a distinctly different output than either genre alone.

PopRockHip-Hop JazzDnBHouse ClassicalAmbientTrap FolkMetalR&B + 18 more
Section C

Style Influences

Influences the style prompt. Free text for vocal type, BPM, instruments, production preferences, and artist references. When artists are mentioned, the system decodes their sonic DNA into production descriptors (see Artist Reference Protocol).

Example: "male vocals, 120 BPM, analog synths, reverb-heavy, Radiohead-like guitar textures"

Section D

Lyric Input

Affects lyrics output ONLY — does not influence style or exclusions. If the user already has lyrics or lyric ideas, they go here. Gemini preserves the user's creative intent while refining prosody and adding Suno metatags.

If left empty, Gemini composes original lyrics from the other sections.

Section E

Exclusions

Hard constraints applied to relevant outputs. Style exclusions go into the negative prompt; content exclusions (e.g. "no profanity") affect lyrics; vocal exclusions (e.g. "no female voices") affect both style and exclusions.

Example: "no female voices, no autotune, no trap beats, no profanity"

How Builder Input Is Compiled

The frontend assembles non-empty sections into a delimited text block:

---INSPIRATION---
A euphoric summer anthem about freedom and road trips

---GENRES---
Pop, House, Funk

---STYLE_INFLUENCES---
male and female vocal duet, 124 BPM, Avicii-style progressive builds, slap bass

---LYRICS_INPUT---
(empty — omitted from compilation)

---EXCLUSIONS---
no autotune, no mumble rap

This string is sent as the input field to the API. The Gemini prompt template (Builder Preamble) instructs the model exactly how to interpret each delimited section.

Artist Reference Protocol

Critical Rule

When users mention real artist names (e.g. "Avicii", "Radiohead", "Billie Eilish"), the system decodes their sonic DNA into generic production descriptors but never outputs the actual artist name in any production-facing section (styles, exclude, or lyrics).

Style Prompt Priority Structure

The style prompt is the most important output — it controls everything about how Suno renders the track. Gemini is instructed to structure it in this exact priority order:

  1. Genre Foundation (2–4 terms) — Primary genre + subgenre qualifiers with era-specific context
  2. Vocal Specification (3–5 terms) — Gender, delivery style, technique, vocal processing
  3. Instrumentation & Arrangement (4–8 terms) — Named instruments with tonal adjectives, rhythmic elements, prominence order
  4. Production & Sonic Character (3–5 terms) — Mix techniques, spatial qualities, analog/digital character
  5. Tempo, Key & Energy (2–3 terms) — Explicit BPM, musical key, energy descriptor
melancholic indie folk meets neo-soul, breathy female soprano with intimate
whisper-to-belt dynamics, warm Rhodes electric piano, fingerpicked nylon
guitar, upright bass, brushed snare, subtle plate reverb on vocals,
analog tape saturation, 3D spatial imaging, pristine studio mix, 78 BPM,
D minor, late-night contemplative groove, radio edit

Plain Lyrics Extraction

The parser strips metatags to produce clean lyrics for display and album cover input:

  • Removed entirely: [Title: ...], [Style: ...], [End], instrumental cues like [Drum Fill], [Synthesizer Solo]
  • Converted: [Verse 1: Low-Register, Intimate] → Verse 1:
  • Inline cleanup: Parenthetical backing vocals (harmony note) and bracketed cues removed from within lyric lines
  • Whitespace: Collapsed to max 2 consecutive blank lines

Album Cover Generation

The album cover is a two-stage process: AI artwork generation (Imagen 4.0 Ultra) followed by canvas-based typography composition in the browser.

Stage 1: Artwork Generation

The plain lyrics (max 2,000 chars) and style prompt (max 500 chars) are injected into a specialised Imagen prompt that instructs the model to generate:

  • Massive, surreal central subject inspired by the lyrics
  • Natural landscape background with cinematic depth
  • Colour theme adapted to the emotional core of the lyrics
  • Dynamic textures (iridescent glass, bioluminescence, liquid gold, ethereal mist)
  • Top ~15% slightly darker (for title overlay)
  • Bottom ~12% slightly darker (for branding overlay)
  • Absolutely no text, typography, logos, or watermarks

Output: 1:1 aspect ratio, returned as base64-encoded image.

Stage 2: Typography Composition

A 1400×1400px canvas composites the AI artwork with two overlay layers:

Top Layer — Song Title

  • Dark gradient scrim (top 22% of canvas)
  • Song title in Bitcount Single font
  • Metallic silver gradient text with drop shadow
  • Auto-sized to fit 88% canvas width (min 28px, max 96px)

Bottom Layer — ADRIFT BEATS Branding

  • Dark gradient scrim (bottom 22%)
  • Central waveform visualisation (60 bars, cyan glow, heights from real logo)
  • Cyan dotted lines on left/right sides
  • "ADRIFT BEATS" text in Science Gothic font at 42px, weight 900
  • White-to-cyan gradient with 15px glow shadow

The composed image is exported as a PNG data URL for download.

Why editing matters: Because the album cover reads the song title and styles from the live result state, editing the title or styles before generating the cover ensures the artwork and typography reflect your latest changes.

Editing & Saving

Inline Output Editing

Every output section (Song Title, Lyrics, Style Prompt, Excluded Styles) has an Edit button in its header. Clicking it swaps the display for an inline editor. Changes are committed with Save and immediately propagate to all consumers — including the album cover generator, which will use the updated title, styles, and lyrics on next generation.

Title sync: When the song title is edited, SunoBrain automatically updates the [Title: ...] tag inside the optimized lyrics so they stay in sync.

Saved Styles Library

In Builder mode, the Style Prompt output card shows a Save button. Saving stores the style with:

  • Unique name (enforced, case-insensitive)
  • Full style prompt text
  • Exclude styles that accompanied it
  • Genres that were selected at generation time
  • Character count and creation timestamp

The Saved Styles Drawer in the Builder panel lets you browse, expand, rename, delete, and inject any saved style into the Style Influences field for reuse in new generations.

Two-step Flow (Builder or Theme)

Step 1: Draft Generation
Input sent with builder_draft or theme_draft mode. Gemini returns raw lyrics only (no metatags, no analysis, no styles).
User Review & Edit
Draft displayed in an editable textarea. User refines lyrics, adjusts structure, adds or removes content.
Step 2: Optimization
Edited draft sent with optimize_draft mode. Gemini preserves the user's words while adding metatags, generating style prompt, exclusions, and analysis. Full blueprint returned.
